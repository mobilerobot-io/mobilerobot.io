Joystick Controls for Mobile Robot

This was a simple but very cool project. Setup an Arduino Nano with a Joystick inputs with an RF output back to a control station (or the vehicle if control software resides on vehical).

Requirements

Develop a wireless Remote Control capable of controlling (i.e. send inputs to control software) able to control the vehicles motion.

The Joystick must be able to communicate wirelessley.

  • RF - required and PoC is complete
  • Wifi optional and desired
  • Blue Tooth optional and desired

Communications

The communication software will be abstracted away from the underlying transmission mechanism. That is, that communication software that sends data from Joystick over RF, will not change if RF is changed to Wifi, BlueTooth, Serial / UART / USB or any other mechanism.

That is the communication layer will understand the TLV and TLString protocols, provide the application.

Arduino I/O

Arduino recieves inputs from the Joystick, those inputs can be sent in one of two modes: polling or event driven. Polling sends data every delta period, where event driven sends only changes.
